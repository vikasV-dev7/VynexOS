# VynexOS v0.4.1 Release Notes

## Engineering Stabilization
This release pauses architectural feature development to fortify the foundation of the OS. With the upcoming introduction of physical hardware drivers (SDL2/OpenGL) in Version 0.5.0, a robust regression testing harness is mandatory. 

### Features & Improvements
- **Automated Regression Verification:** The CMake/CTest ecosystem has been unified. The command `ctest -C Debug --output-on-failure` now serves as the primary verification gateway for the entire codebase.
- **Hardware Abstraction Testing:** Created the dedicated `HALTests` executable. This suite strictly tests Storage, Network, and Hardware Clock abstractions, ensuring bounds checking, loopback delivery, and RAII lifecycle determinism.
- **Configurable Logging Verbosity:** Extended the `vynexos::core::[[ILogger]]` interface with `set_log_level()`. Automated test suites now automatically default to `INFO`, entirely suppressing noisy `TRACE` and `DEBUG` outputs.
- **Idling & Stability:** Removed early demonstration scaffolding from `CompositionRoot` that simulated continuous user input and phantom notifications. The runtime now correctly idles when no events are actively routed.
