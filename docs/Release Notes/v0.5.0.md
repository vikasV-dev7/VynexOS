# VynexOS v0.5.0 - Physical Driver Layer

**Release Date**: 2026-07-18

## Overview
Version 0.5.0 introduces the **Physical Driver Layer**, finalizing the transition from software-mocked hardware abstractions to concrete GPU-accelerated environments using SDL2. This release completely stabilizes the rendering pipeline, physical input translation, and precise frame timing logic without compromising the strictly isolated headless testing capabilities.

## Major Changes & Architecture Improvements
- **Hardware Rendering (SDL2)**: Replaced mock drivers with `SDL2DisplayBackend` for hardware-accelerated ARGB8888 rendering mapping to standard VynexOS pipelines.
- **Strict PIMPL Abstraction**: Enforced pointer-to-implementation (`SDLContext`) across the desktop boundaries. SDL headers and macros (like `SDL_main`) are strictly prevented from polluting the public VynexOS ABI.
- **Input Hardware Integration**: `SDL2InputDriver` now securely translates synthetic hardware polls (`SDL_KEYDOWN`, `SDL_MOUSEMOTION`) into deterministic VynexOS polymorphic topics (`HAL_INPUT_KEY`, `HAL_INPUT_MOUSE`) on the decentralized `IEventBus`.
- **Frame Synchronization**: Introduced `vynexos::core::FrameClock` to aggressively lock background polling into a stable 60 FPS update cycle.

## Test Infrastructure & Quality Assurance
- Implemented `SDLIntegrationTests`, bringing overall test suite count to four fully resilient pipelines.
- Implemented mechanical runtime testing for deterministic `FrameClock` chronometer analysis.
- Verified 100% CTest pass rate locally and validated RAII stability.
- Formally integrated `docs/Technical Debt.md` to track architectural concessions (e.g. TD-0001).

## Upgrading
No breaking API changes to the VynexOS core service interfaces. Modules relying exclusively on `IEventBus` or `IDisplayAbstraction` will automatically benefit from the physical layer without recompilation.
