# VynexOS Version 0.6.0 Release Notes

## Overview
Version 0.6.0 is a major stabilization and architectural validation milestone. This release formally proves that VynexOS's headless, dependency-injected mock-services architecture can survive under load, maintain strict deterministic event processing, and render complex multi-layered UI logic safely without allocating dynamic memory on the critical rendering path.

## Major Features
- **Runtime Lifecycle Completion**: `CompositionRoot` now natively handles lifecycle teardown via `is_shutdown_requested` on the `IInputDriver`, initiating a strict reverse-initialization teardown process that fully cleans up all services.
- **Native Software Rendering**: Completely purged the legacy WindowBuffer system. The `BasicDesktopShell` now fully implements the formal `ISurface` rendering pipeline directly against the backend FrameBuffer.

## Architecture Improvements
- **Event Bus Isolation**: Event Bus messaging no longer dictates the startup/shutdown lifecycle. The hardware abstraction layer naturally polls for completion.
- **Queue-Based Event Determinism**: Replaced asynchronous hardware polling (`SDL_GetMouseState`) with internal deterministic mouse button queue state tracking. This fully isolates the application logic from asynchronous physical driver race conditions.

## Performance Improvements
- **Zero-Allocation Rendering**: Re-architected `BasicCompositor` by elevating the master frame buffer to class scope, eliminating the 8.29 MB per-frame dynamic allocation overhead.
- **MSVC Debug Iterator Bypass**: Successfully profiled and removed severe `std::vector` bounds-checking overhead inside nested composition loops by utilizing `std::vector::data()` and `std::memcpy`.
  - **Results**: Average render time reduced from 171.1ms to 23.3ms (an 86% reduction), increasing actual observed framerates by 338% while preserving total mathematical correctness.

## Bug Fixes
- **Ghost Windows**: Fixed a double pre-multiplication artifact inside `BlendPixel` causing background UI elements to bleed through 100% opaque windows.
- **Launcher Disappearing**: Fixed z-ordering logic where rapid, multi-threaded mouse clicks would incorrectly lower active taskbar windows.
- **Encapsulation Violation (INC-0004)**: Patched `test_runtime_bug.cpp` which was directly accessing private internals of `CompositionRoot`.

## Regression Coverage
- Over 60,000 continuous compositor frame runs via MSVC runtime analysis.
- 10,000 rapid window z-order permutations testing iterator invalidation safety.
- Full burst-input test validating strict down/up deterministic ordering.

## Known Limitations
- The `BasicFileExplorer` tabs remain static/non-interactive mock UI elements pending the VFS (Virtual File System) architecture slated for future sprints.
- Native fonts are currently rendered using a minimal `font8x8` header; full TrueType font parsing is not yet implemented.

## Future Roadmap
With the runtime and display pipelines now fully stabilized and performing safely, Version 0.7.0 will shift focus towards process management, Virtual File Systems (VFS), and more expansive desktop application logic.
